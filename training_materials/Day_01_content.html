Semiconductor terminology

  • DUT (Device Under Test) / UUT (Unit Under Test) อุปกรณ์ที่ถูกทดสอบ
  • DPS (Device Power Supplies) แหล่งจ่ายไฟเลี้ยงให้ DUT
  • PMU (Parametric Measurement Unit) อุปกรณ์สำหรับวัด DC characteristics ของ DUT
  • PPMU (Per Pin Measurement Unit) PMU ที่ต่อกับขาของ DUT
  • DIB (Device Interface Board) / Load Board ส่วนเชื่อมต่อกับ DUT
  • HANDLER อุปกรณ์หยิบจับชิป (Die) ใส่ซ็อคเก็ต
  • PIB (Probe Interface Board) โพรบที่เชื่อมต่อเวเฟอร์
  • Probe Card แผงวงจรที่ต่อกับโพรบ
  • PROBER อุปกรณ์เลื่อนชิป (Die) ไปที่โพรบ
  • VCC (+ ICC) ไฟเลี้ยง TTL
  • VDD (+ IDD) ไฟเลี้ยง CMOS
  • VSS ทางกลับของไฟเลี้ยง
  • VIH / VIL แรงดันเข้าสำหรับ logic 1 / 0
  • VOH / VOL แรงดันออกสำหรับ logic 1 / 0
  • IOH / IOL กระแส source / sink สำหรับ logic 1 / 0
  • IIH / IIL กระแส leakage สำหรับ logic 1 / 0
  • IOZH / IOZL กระแส leakage ในสถานะ high-Z สำหรับ logic 1 / 0

Digital logics vs electrical signals



Logic Voltage
Out = 1 \(V_O\) > \(V_{OH}\)
Out = 0 \(V_O\) < \(V_{OL}\)
In = 1 \(V_I\) > \(V_{IH}\)
In = 0 \(V_I\) < \(V_{IL}\)
Logic Transistor Voltage
Out = 1 OFF \(V_{load}\) = \(V_{DD} - I_{load} * R\)
Out = 0 ON \(V_{load}\) ≅ 0 = \(I_{load} * R\)

Why LabVIEW?

Hardware interfacing



Virtual Instrument → UI



NI MAX

Measurement & Automation Explorer ใช้ตั้งค่าและตรวจสอบเครื่องมือวัด/DAQ/PXI

Why LabVIEW for ATE?

NI-DAQmx

  • NI instrument driver that controls DAQ systems
  • Library of VIs, functions, classes, attributes, and properties to configure, control, stream data



NI-VISA

  • Virtual Instrument Software Architecture
  • Instrument driver and software API used to configure, program, and control instruments
  • Interfaces: USB, Ethernet, GPIB, Serial.


NI-PXI

  • HW/SW platform built for test automation
  • Synchronize across multiple modules or chassis
  • PXI embedded controller = PC inside PXI


LabVIEW

Front panel: ส่วนหน้าจอ

  • ประกอบด้วย Controls (รับคำสั่ง/ข้อมูล) และ Indicators (แสดงผล)
  • ทำงานแบบ interactive


Block diagram: ส่วนโค้ด

  • GUI style → drag & drop แล้ว connect
  • Execution → dataflow programming
    • (รอ) ข้อมูลเข้า → ประมวลผลและส่งต่อ
    • ทำงานแบบ parallel loop ได้
  • แนะนำไล่จาก ซ้าย ไป ขวา

LabVIEW terminology

  • Data types:
    • Numeric: สีส้ม (DBL/SGL), สีน้ำเงิน (INT/UINT/enum)
    • Boolean: สีเขียว
    • Array: เส้นหนา (1D), เส้นคู่ (2D)
    • String และ Cluster: สีชมพู
  • Control flow:
    • While: ทำงานตามเงื่อนไข
    • For: N รอบ หรือตามจำนวนข้อมูล
    • Case structure: if-else, switch-case
  • SubVI:
    • Connector Pane: Required, Recommend, Optional
  • Timing:
    • Loop + delay: ง่ายแต่ไม่แม่น
    • Timed loop: ความแม่นยำ/ความละเอียดทางเวลา
    • Block waiting: รอข้อมูลจาก HW
  • Data passing:
    • Shift register: ส่งข้อมูลข้าม loop
    • Local variable: แชร์การเข้าถึง
    • กลุ่ม synchronized function
  • UI programming:
    • Event structure: callback ของ UI event
    • Property node: อัพเดท UI

ADS1115 schematics


ADS1115 operations

Multiplexer (MUX):

  • Singled-ended, differential

Programmable Gain Amplifier (PGA):

  • \(LSB = \frac{FSR}{2^{16}}\)

I2C write → ADS1115

  • I2C bus: half-duplex, master-slave (7-bit dev address + R/W bit)
  • I2C device: registers for control/status/data

I2C read ← ADS1115


Programming ADS1115

Configure modes

  • write(device address+W) + address pointer + (MSB/LSB) payload

Read data

  • write(device address+W) + address pointer
  • read(device address+R) + 2-byte payload

Device address (7bit) ขึ้นกับขา ADDR

  • GND (1001000) หรือ VDD (1001001)

Address Pointer

Register Address Function
Conversion 00b ข้อมูล 16 บิตล่าสุด
Config 01b ตั้งโหมด เลือกอินพุต ตั้งอัตราสุ่ม กำหนดย่าน ตั้ง comparator
Low Thres 10b กำหนดเงื่อนไขช่วงต่ำ → ALERT
High Thres 11b กำหนดเงื่อนไขช่วงสูง → ALERT
